ddl-auto

create : 어플리케이션 시작시 DB 기존 테이블을 삭제하고 우리가 만든 
엔티티 기반으로 테이블을 생성해준다

create-drop : 테이블을 생성하지만, 어플리케이션이 종료 되면 다시 삭제를 해준다
                   => 테스트 환경에서 많이 사용한다 (일단 해보고 app 이 끝나면 삭제해준다)

update : 기존 DB 스키마를 엔티티 클래스에 맞게끔 업데이트 해준다
	  => 기존에 데이터의 손실을 유발하지 않고 추가되거나, 삭제되는 것을 반영해준다

none : DB 의 스키마를 변경하지 않는다. 즉 기존의 데이터베이스를 유지 할 수 있다
